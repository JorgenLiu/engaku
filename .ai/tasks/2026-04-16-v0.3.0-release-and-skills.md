## Background

v0.2.0 was tagged at commit `63a3876`, but three CI-fix commits landed after
the tag and were never published. Additionally, three post-v0.2.0 features
landed without a release:

1. **SubagentStart hook** — `cmd_inject.py` now handles `SubagentStart` events
   and emits `hookSpecificOutput`; the reviewer agent template includes the new
   hook.
2. **`proactive-initiative` skill** — a fourth bundled skill was added to
   `cmd_init.py` and the templates directory.
3. **CI/CD workflows + package-data fix** — `.github/workflows/ci.yml` and
   `publish.yml` were created; `pyproject.toml` package-data was fixed so all
   template subdirectories are bundled correctly.

This plan brings the repo up to date (metadata, changelog, housekeeping) and
adds two new skills (`mcp-builder` and `doc-coauthoring`) before tagging and
publishing v0.3.0.

## Design

### Skill adaptation strategy

The two new skills are adapted from `https://github.com/anthropics/skills/tree/main/skills`.
The anthropics versions rely on bundled reference file trees and a code execution
sandbox. The engaku versions must be self-contained Markdown files that work in
VS Code Copilot without bundled scripts.

**mcp-builder**: Keep the 4-phase workflow structure (Research → Implement →
Review → Evaluate). Replace all `[Read ./reference/node_mcp_server.md]`
directives with explicit `WebFetch` instructions pointing to the official MCP and
SDK README URLs. Drop the evaluation XML format — it assumes Claude Code subagents.

**doc-coauthoring**: Keep the 3-stage workflow (Context Gathering → Refinement &
Structure → Reader Testing). Strip clauses that reference Claude-Code-only tools
(file system ops, subagents). Frame around VS Code Copilot chat context.

### Metadata cleanup

- `.github/copilot-instructions.md` line `Distribution: … PyPI later.` is stale.
  Replace with the current dual-install line.
- `.ai/overview.md` still says "3 stubs for skills" and lists 3 skill names. Update
  to reflect 4 skills and the SubagentStart hook.
- `CHANGELOG.md` has no entry for anything after v0.2.0. Add a proper v0.3.0 block.

### Package-data

`pyproject.toml` already uses `"templates/skills/*/*.md"` which covers any new
skill subdirectory automatically. No change needed.
